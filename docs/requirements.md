# CommissionCalc 需求文档

## 项目概述

| 项目属性 | 说明 |
|----------|------|
| 项目名称 | CommissionCalc - 绩效提成计算系统 |
| 开发语言 | Python 3.8+ |
| 运行平台 | Windows |
| 界面类型 | GUI（Tkinter） |
| 数据格式 | 文本导入、Excel导出 |

## 功能需求

### 核心功能

#### 1. 粘贴文本导入业绩

- 支持粘贴制表符分隔的文本数据
- 自动识别"姓名"列和业绩列（业绩列名可配置，默认"累计业绩"）
- 支持从Excel直接复制粘贴
- 导入前可预览识别结果，显示匹配状态

#### 2. Excel结果导出

**基础导出功能**：
- 导出详细的提成明细报表（数值格式，无文本告警）
- 包含姓名、业绩、身份、组别、各项提成、奖金、总提成

**高级导出功能**：
- 支持粘贴名单（姓名列表）自定义导出顺序
- 导出报表顺序与粘贴名单顺序一致
- 名单中不存在的人员数据补0
- 名单顺序可保存到配置，下次导出可直接使用

#### 3. 人员配置管理

- 配置人员身份（总主管、组长、成员）
- 配置人员所属组别
- 自动修复组成员关系不一致（启动时根据人员group_id同步组的members）
- 保存配置信息，程序重启后自动加载

#### 4. 提成规则配置

- 个人提成阶梯配置（可动态添加/删除）
- 团队提成阶梯配置（组长专用，含达标线配置）
- 总主管达标线配置（独立于团队达标线）
- 管理提成金额配置
- 高业绩奖金阶梯配置（可动态添加/删除）
- 所有配置保存后程序重启自动加载

#### 5. 自动提成计算

- 根据配置规则自动计算各类提成
- 支持多种提成类型组合计算
- 详细计算日志便于排查问题（输出到log/YYYY-MM-DD.log）

## 数据模型

### 人员信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | String | 人员姓名（唯一标识） |
| 个人业绩 | Float | 当期业绩金额（默认0） |
| 身份 | Enum | 总主管（仅1位）/组长/成员 |
| 组别 | String | 所属组别ID（总主管可为空） |

### 组别信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 组名 | String | 组别名称 |
| 组长 | String | 该组组长ID |
| 成员 | List | 该组成员ID列表 |

## 提成计算规则

### 1. 个人业绩提成

**适用对象**：所有人员（总主管、组长、成员）

**计算方式**：全额计算（匹配阶梯后，全部业绩按该阶梯比例计算）

| 业绩范围 | 提成比例 |
|----------|----------|
| 0 ~ 3000元 | 0% |
| 3000元以上 | 20% |

**计算示例**：

```
业绩 2500元 → 2500 × 0% = 0元
业绩 3000元 → 3000 × 20% = 600元
业绩 5000元 → 5000 × 20% = 1000元
业绩 15000元 → 15000 × 20% = 3000元
```

### 2. 组长团队业绩提成

**适用对象**：组长

**计算依据**：
- 累加组内达标业绩（业绩 >= 达标线，默认3000元）
- 包含组长自己的业绩
- 未达标人员业绩不计入

**计算方式**：全额计算（匹配阶梯后，全部团队业绩按比例计算）

| 团队业绩范围 | 提成比例 |
|--------------|----------|
| 0 ~ 10000元 | 0% |
| 10000 ~ 50000元 | 10% |
| 50000元以上 | 20% |

**计算示例**：

```
组长 20000(达标) + 成员A 40000(达标) + 成员B 2000(未达标)
→ 团队业绩 = 60000元 → 60000 × 20% = 12000元

组长 20000(达标) + 成员A 2000(未达标) + 成员B 2000(未达标)
→ 团队业绩 = 20000元 → 20000 × 20% = 4000元

全组未达标 → 团队业绩 = 0元 → 0元提成
```

### 3. 总主管团队业绩提成

**适用对象**：总主管

**计算依据**：
- 累加所有达标业绩（业绩 >= 达标线，默认50000元）
- 包含总主管自己的业绩
- 未达标人员业绩不计入
- **达标线独立配置**，与组长达标线分开

**计算方式**：全额计算（匹配阶梯后，全部团队业绩按比例计算）

| 团队业绩范围 | 提成比例 |
|--------------|----------|
| 0 ~ 50000元 | 0% |
| 50000元以上 | 10% |

**注意**：总主管达标线（50000元）与组长达标线（3000元）独立配置。

### 4. 组长管理提成

**适用对象**：仅组长

**计算方式**：固定金额

**规则**：组内成员数（不包括组长） × 每人金额（默认100元），不要求成员达标

**计算示例**：

```
组内2名成员 → 2 × 100 = 200元
组内5名成员 → 5 × 100 = 500元
```

### 5. 高业绩奖金

**适用对象**：所有人员（总主管、组长、成员）

**计算方式**：不累加，取最高梯度奖金

| 业绩阈值 | 奖金金额 |
|----------|----------|
| ≥ 20000元 | 500元 |
| ≥ 30000元 | 1000元 |
| ≥ 50000元 | 2000元 |

**计算示例**：

```
业绩 18000元 → 未达阈值 = 0元
业绩 20000元 → 达2万 = 500元
业绩 25000元 → 达2万 = 500元
业绩 30000元 → 达3万 = 1000元
业绩 35000元 → 达3万 = 1000元
业绩 50000元 → 达5万 = 2000元
业绩 60000元 → 达5万 = 2000元
```

### 总提成公式

```
总提成 = 个人业绩提成 + 团队业绩提成 + 管理提成 + 高业绩奖金
```

**完整示例**：

| 人员 | 个人业绩 | 身份 | 个人提成 | 团队提成 | 管理提成 | 奖金 | 总提成 |
|------|----------|------|----------|----------|----------|------|--------|
| 总主管 | 50000 | 总主管 | 10000 | 12000 | 0 | 2000 | 24000 |
| 组长A | 20000 | 组长 | 4000 | 4000 | 200 | 500 | 8700 |
| 成员A1 | 40000 | 成员 | 8000 | 0 | 0 | 1000 | 9000 |
| 成员A2 | 2000 | 成员 | 0 | 0 | 0 | 0 | 0 |

## 数据格式

### 导入格式（粘贴文本）

**必需列**：
- 姓名（固定列名）
- 业绩（列名可配置，默认"累计业绩"）

**格式示例**：

```
姓名    累计业绩
张三    5000
李四    3000
王五    15000
```

说明：制表符分隔，支持从Excel直接复制粘贴。

### 导出格式

**标准输出列**：
- 姓名、业绩、身份、组别
- 个人提成、团队提成、管理提成、高业绩奖金、总提成

**自定义顺序导出**：
- 可粘贴姓名名单指定导出顺序
- 名单中不存在的人员数据补0
- 顺序可保存到配置，下次导出复用

**格式示例**：

```
| 姓名 | 业绩 | 身份 | 组别 | 个人提成 | 团队提成 | 管理提成 | 奖金 | 总提成 |
| 张三 | 5000 | 组长 | A组 | 1000 | 800 | 200 | 0 | 2000 |
| 李四 | 3000 | 成员 | A组 | 600 | 0 | 0 | 0 | 600 |
```

## 配置需求

### 提成规则配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 个人提成阶梯 | 0-3000(0%), 3000+(20%) | 可动态增删 |
| 团队提成阶梯 | 0-10000(0%), 10000-50000(10%), 50000+(20%) | 可动态增删 |
| 组长达标线 | 3000元 | 团队业绩累加阈值 |
| 总主管达标线 | 50000元 | 独立配置，与组长分开 |
| 管理提成每人金额 | 100元 | 组长管理提成 |
| 高业绩奖金阶梯 | 2万(500), 3万(1000), 5万(2000) | 可动态增删 |

### 配置存储

- 存储方式：JSON文件
- 配置目录：`config/`
- 配置文件：`settings.json`（提成规则）、`people.json`（人员配置）
- 导出顺序：`export_order.json`（保存的导出名单顺序）
- 启动时自动加载，界面修改后手动保存生效

## 界面需求

### 主界面（提成计算标签页）

**数据操作区**：
- 粘贴文本导入按钮
- 计算提成按钮
- 导出结果按钮

**数据预览区**：
- 显示导入的业绩数据（姓名、业绩、身份、组别）

**结果展示区**：
- 显示计算结果明细表格（各项提成、奖金、总计）

### 人员管理标签页

**人员列表**：
- 显示所有人员（姓名、身份、组别）
- 添加/编辑/删除人员按钮
- 保存配置按钮

**配置规则**：
- 总主管只能有一位
- 组长必须分配到组
- 成员必须分配到组

### 规则配置标签页

**个人提成阶梯**：
- 表格显示（下限、上限、比例）
- 添加/编辑/删除按钮

**团队提成阶梯**：
- 表格显示（下限、上限、比例）
- 组长达标线输入框
- 总主管达标线输入框（独立配置）
- 添加/编辑/删除按钮

**管理提成**：
- 每人管理提成金额输入框

**高业绩奖金**：
- 表格显示（阈值、金额）
- 添加/编辑/删除按钮

**操作按钮**：
- 保存配置
- 重置默认

### 分层架构

```
src/
├── models/        # 数据模型层（Person, Group, Config）

├── repositories/  # 数据访问层（config_repo, excel_repo, people_repo）
├── ui/            # 界面层（main_window, dialogs）
```
