# CommissionCalc 需求文档

## 项目概述

**项目名称**: CommissionCalc - 绩效提成计算系统  
**开发语言**: Python 3.8+  
**运行平台**: Windows  
**界面类型**: GUI（Tkinter）  
**数据格式**: Excel（导出）、文本（导入）

## 功能需求

### 核心功能

1. **粘贴文本导入业绩**
   - 支持粘贴制表符分隔的文本数据
   - 自动识别姓名列和业绩列（列名可配置）
   - 支持Excel复制粘贴

2. **Excel结果导出**
   - 导出详细的提成明细报表（数值格式）
   - 包含各项提成、奖金和总提成

3. **人员配置管理**
   - 配置人员身份（总主管、组长、成员）
   - 配置人员所属组别
   - 自动修复组成员关系不一致
   - 保存配置信息，程序重启后自动加载

4. **提成规则配置**
   - 可配置的提成阶梯比例
   - 可配置的达标线（团队业绩）
   - 可配置的管理提成金额
   - 可配置的高业绩奖金阈值
   - 保存配置信息，程序重启后自动加载

5. **自动提成计算**
   - 根据配置规则自动计算各类提成
   - 支持多种提成类型的组合计算
   - 详细计算日志便于排查问题

## 数据需求

### 人员信息

- **姓名**: 人员姓名（唯一标识）
- **个人业绩**: 该人员当期业绩金额
- **身份**: 三种身份类型
    - 总主管：仅一位，管理所有组长
    - 组长：管理一个组
    - 成员：归属于某个组
- **组别**: 所属组别（总主管可为空）

### 组别信息

- **组名**: 组别名称
- **组长**: 该组的组长ID
- **成员**: 该组的成员列表

## 提成计算规则

### 1. 个人业绩提成

**适用对象**: 所有人员（总主管、组长、成员）

**计算方式**: 全额计算（达到阶梯后，全部业绩按该阶梯比例计算）

**阶梯规则**:

| 业绩范围 | 提成比例 |
|---------|---------|
| 0-3000元 | 0% |
| 3000元以上 | 20% |

**计算示例**:
- 业绩2500元 → 2500 × 0% = 0元
- 业绩3000元 → 3000 × 20% = 600元
- 业绩5000元 → 5000 × 20% = 1000元
- 业绩15000元 → 15000 × 20% = 3000元

### 2. 团队业绩提成

**适用对象**: 组长、总主管

**计算方式**: 达标业绩累加（>=达标线，默认3000元），全额计算

**计算依据**:
- **组长**: 组内所有成员达标业绩累加（包括组长自己）
- **总主管**: 所有达标人员业绩累加（包括总主管自己）
- 未达标人员业绩不计入团队业绩
- 达标线可在规则配置中修改

**阶梯规则**:

| 团队业绩范围 | 提成比例 |
|-------------|---------|
| 0-10000元 | 0% |
| 10000-50000元 | 10% |
| 50000元以上 | 20% |

**计算示例**:
- 组长业绩20000(达标) + 成员A业绩40000(达标) + 成员B业绩2000(未达标) → 60000 × 20% = 12000元
- 组长业绩20000(达标) + 成员A业绩40000(达标) → 60000 × 20% = 12000元
- 所有成员业绩未达标 → 0元

### 3. 组长管理提成

**适用对象**: 仅组长

**计算方式**: 固定金额（成员数量 × 每人金额）

**规则**: 每管理一名成员（不包括自己），获得100元管理提成（不要求成员达标）

**计算示例**:
- 组内有2名成员 → 2 × 100 = 200元
- 组内有5名成员 → 5 × 100 = 500元

### 4. 高业绩奖金

**适用对象**: 所有人员（总主管、组长、成员）

**计算方式**: 不累加，取最高梯度奖金

**阶梯规则**:

| 业绩阈值 | 奖金金额 |
|---------|---------|
| 达到2万 | 500元 |
| 达到3万 | 1000元 |
| 达到5万 | 2000元 |

**计算示例**:
- 业绩18000元 → 未达到任何阈值 = 0元
- 业绩20000元 → 达到2万 = 500元
- 业绩25000元 → 达到2万 = 500元
- 业绩30000元 → 达到3万 = 1000元
- 业绩35000元 → 达到3万 = 1000元
- 业绩50000元 → 达到5万 = 2000元

## 总提成计算

**总提成 = 个人业绩提成 + 团队业绩提成 + 管理提成 + 高业绩奖金**

**计算示例**:

| 人员 | 个人业绩 | 身份 | 个人提成 | 团队提成 | 管理提成 | 奖金 | 总提成 |
|------|---------|------|---------|---------|---------|------|-------|
| 总主管 | 5000 | 总主管 | 1000 | 2600 | 0 | 0 | 3600 |
| 组长A | 5000 | 组长 | 1000 | 800 | 200 | 0 | 2000 |
| 成员A1 | 5000 | 成员 | 1000 | 0 | 0 | 0 | 1000 |
| 成员A2 | 3000 | 成员 | 600 | 0 | 0 | 0 | 600 |

## 数据格式

### 导入格式（粘贴文本）

**必需列**:
- 姓名（字符串）
- 业绩（数字，列名可配置，默认"累计业绩"）

**示例**:

```
姓名    累计业绩
张三    5000
李四    3000
王五    15000
```

说明：支持制表符分隔格式，可直接从Excel复制粘贴

### 导出格式

**输出列**:
- 姓名
- 业绩
- 身份
- 组别
- 个人提成
- 团队提成
- 管理提成
- 高业绩奖金
- 总提成

**示例**:

```
| 姓名 | 业绩 | 身份 | 组别 | 个人提成 | 团队提成 | 管理提成 | 奖金 | 总提成 |
|------|------|------|------|---------|---------|---------|------|-------|
| 张三 | 5000 | 组长 | A组 | 1000 | 800 | 200 | 0 | 2000 |
```

## 配置需求

### 提成规则配置

1. **个人业绩提成阶梯**
   - 可动态配置阶梯数量和比例
   - 默认：0-3000(0%), 3000+(20%)

2. **团队业绩提成阶梯**
   - 可动态配置阶梯数量和比例
   - 可配置达标线（默认3000元）
   - 默认：0-10000(0%), 10000-50000(10%), 50000+(20%)

3. **管理提成金额**
   - 可配置每人管理提成金额
   - 默认：每人100元

4. **高业绩奖金阶梯**
   - 可动态配置阈值和奖金金额
   - 默认：2万(500), 3万(1000), 5万(2000)

### 配置存储

- 使用JSON文件存储配置
- 配置文件位置：`config/settings.json`
- 程序启动时自动加载配置
- 用户可在界面中修改配置并保存

## 界面需求

### 主界面（提成计算标签页）

1. **数据操作区**
   - 粘贴文本导入按钮
   - 计算提成按钮
   - 导出结果按钮

2. **数据预览区**
   - 显示导入的业绩数据
   - 显示人员配置信息（姓名、业绩、身份、组别）

3. **结果展示区**
   - 显示计算结果明细表格
   - 包含个人提成、团队提成、管理提成、奖金、总计

### 人员管理标签页

1. **人员列表**
   - 显示所有人员（姓名、身份、组别）
   - 添加/编辑/删除人员按钮
   - 保存配置按钮

2. **配置规则**
   - 总主管只能有一位
   - 组长必须分配到组
   - 成员必须分配到组

### 规则配置标签页

1. **个人提成阶梯配置**
   - 表格显示阶梯（下限、上限、比例）
   - 添加/编辑/删除阶梯按钮

2. **团队提成阶梯配置**
   - 表格显示阶梯（下限、上限、比例）
   - 达标线输入框
   - 添加/编辑/删除阶梯按钮

3. **管理提成配置**
   - 每人管理提成金额输入框

4. **高业绩奖金配置**
   - 表格显示奖金阶梯（阈值、金额）
   - 添加/编辑/删除阶梯按钮

5. **操作按钮**
   - 保存配置按钮
   - 重置默认按钮
